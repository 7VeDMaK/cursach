\section{Выбор и обоснование функциональной схемы разрабатываемого алгоритма}

\subsection{Архитектура программного комплекса и маршрутизация данных}

Программная реализация гибридного кодека выполнена на языке программирования Python 3 с использованием микрофреймворка Flask для организации серверной логики и веб-интерфейса. Выбор Python обусловлен наличием обширной экосистемы библиотек для компьютерного зрения и машинного обучения.

Функционально серверная часть приложения (контроллер) разделена на логические блоки, обеспечивающие полный цикл обработки запроса:
\begin{enumerate_num}
    \item \textbf{Инициализация окружения:} При запуске сервера создаются директории для временного хранения загружаемых оригиналов (\texttt{UPLOAD\_FOLDER}) и сгенерированных результатов (\texttt{RESULTS\_FOLDER}). Параллельно в оперативной памяти инициализируются объекты классов \texttt{WaveletCompressor} и \texttt{AIEnhancer}.
    \item \textbf{Обработка пользовательского ввода:} Контроллер принимает HTTP POST-запрос, содержащий графический файл. Файл сохраняется на файловой системе сервера, после чего его абсолютный путь передается в конвейер обработки.
    \item \textbf{Управление потоком данных:} Контроллер последовательно вызывает методы сжатия и нейросетевой реконструкции. Для симуляции реального канала передачи данных сжатое вейвлет-представление (LL-скелет) сохраняется на диск с использованием базового кодирования JPEG (уровень качества 90\%). Это позволяет наглядно оценить физическую экономию дискового пространства.
    \item \textbf{Сбор статистики:} Вычисляются размеры оригинального, сжатого и восстановленного файлов (в килобайтах), а также рассчитывается фактический коэффициент компрессии в процентах. Полученные данные, совместно с путями к изображениям и рассчитанными метриками качества, передаются в HTML-шаблон для рендеринга итоговой страницы.
\end{enumerate_num}

\subsection{Программная реализация вейвлет-кодера}

Модуль декомпозиции и сжатия реализован в классе \texttt{WaveletCompressor} с использованием библиотеки \texttt{PyWavelets} (\texttt{pywt}) и библиотеки компьютерного зрения \texttt{OpenCV} (\texttt{cv2}).

Алгоритм обработки отдельного изображения включает следующие функциональные этапы:
\begin{itemize}
    \item \textbf{Подготовка тензора:} Загруженное изображение конвертируется в массив чисел с плавающей точкой (\texttt{numpy.float32}). Это критически важно для предотвращения потери точности и возникновения ошибок переполнения (overflow) в процессе вычисления сверток.
    \item \textbf{Разделение цветовых каналов:} Изображение разбивается на независимые цветовые плоскости (B, G, R). Двумерное дискретное вейвлет-преобразование \texttt{pywt.dwt2} с базисом \texttt{haar} применяется итеративно к каждому из каналов.
    \item \textbf{Нормализация коэффициентов:} Математическая реализация фильтра Хаара в библиотеке \texttt{PyWavelets} приводит к увеличению значений амплитуды аппроксимирующего поддиапазона (LL) пропорционально коэффициентам фильтра. Для возвращения пикселей к оригинальному диапазону яркости, значения матрицы LL принудительно делятся на 2.0.
    \item \textbf{Квантование и сборка:} Нормализованные матрицы аппроксимации для каждого канала объединяются (\texttt{cv2.merge}). Для устранения вычислительного шума применяется математическое округление (\texttt{np.round}), после чего значения жестко ограничиваются в диапазоне $[0, 255]$ функцией \texttt{np.clip} и приводятся к целочисленному типу \texttt{uint8}.
\end{itemize}
На выходе формируется валидное растровое изображение, геометрические размеры которого ровно в два раза меньше исходных.

\subsection{Интеграция модуля нейросетевого апскейлинга}

Задачу реконструкции высокочастотных деталей выполняет класс \texttt{AIEnhancer}. Для обеспечения максимальной производительности при инференсе нейронной сети используется модуль \texttt{dnn\_superres}, встроенный в библиотеку OpenCV.

Функциональная схема модуля декодера:
\begin{enumerate_num}
    \item \textbf{Загрузка весов модели:} При инициализации класса алгоритм проверяет наличие файла сериализованного графа модели \texttt{EDSR\_x2.pb}. В случае его отсутствия запускается автоматическое скачивание актуальных весов из удаленного репозитория.
    \item \textbf{Инициализация суперразрешения:} Создается объект \texttt{DnnSuperResImpl}, в который загружается вычислительный граф. Модели явно задается архитектура (\texttt{edsr}) и коэффициент масштабирования (\texttt{scale = 2}).
    \item \textbf{Инференс:} Сжатое изображение передается в метод \texttt{upsample}. Библиотека OpenCV автоматически выделяет тензоры, проводит данные через все сверточные слои EDSR и выполняет финальную субпиксельную перестановку (PixelShuffle), возвращая увеличенное изображение.
    \item \textbf{Постпроцессинг геометрии:} Ввиду особенностей целочисленного деления при вейвлет-декомпозиции нечетных разрешений, на этапе восстановления возможна размерная нестыковка (микро-сдвиги на 1 пиксель). Алгоритм выполняет программную валидацию геометрии: если размер выходного тензора не совпадает с оригинальным, применяется корректирующее интерполирование (\texttt{cv2.resize}).
\end{enumerate_num}

\subsection{Реализация алгоритма метрического контроля}

Процесс объективной оценки качества реконструкции локализован в отдельном программном скрипте и базируется на двух функциях.

Расчет пикового отношения сигнала к шуму (PSNR) выполняется стандартными средствами OpenCV (\texttt{cv2.PSNR}), которые эффективно обрабатывают многоканальные (BGR) изображения.

Для вычисления индекса структурного сходства (SSIM) применяется функция \texttt{structural\_similarity} из библиотеки \texttt{scikit-image} (\texttt{skimage.metrics}). Поскольку алгоритм вычисления базового SSIM оперирует изменениями локальной контрастности и дисперсии, перед передачей тензоров в функцию оригинальное и восстановленное изображения в обязательном порядке преобразуются из цветового пространства BGR в градации серого (\texttt{cv2.COLOR\_BGR2GRAY}).

Интеграция данных модулей формирует законченную функциональную схему гибридного кодека, реализующую полный цикл от загрузки файла до метрической верификации нейросетевой реконструкции.
