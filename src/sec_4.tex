\section{Выбор и обоснование функциональной схемы разрабатываемого алгоритма}

\subsection{Архитектура программного комплекса и маршрутизация данных}

Программная реализация гибридного кодека выполнена на языке программирования Python 3 с использованием микрофреймворка Flask для организации серверной логики и веб-интерфейса. Выбор Python обусловлен наличием обширной экосистемы библиотек для компьютерного зрения и машинного обучения.

Функционально серверная часть приложения (контроллер) разделена на логические блоки, обеспечивающие полный цикл обработки запроса:

1 Инициализация окружения: при запуске сервера создаются директории для временного хранения загружаемых оригиналов (UPLOAD\_FOLDER) и сгенерированных результатов (RESULTS\_FOLDER). Параллельно в оперативной памяти инициализируются объекты классов WaveletCompressor и AIEnhancer.

2 Обработка пользовательского ввода: контроллер принимает HTTP POST-запрос, содержащий графический файл. Файл сохраняется на файловой системе сервера, после чего его абсолютный путь передается в конвейер обработки.

3 Управление потоком данных: контроллер последовательно вызывает методы сжатия и нейросетевой реконструкции. Для симуляции реального канала передачи данных сжатое вейвлет-представление (LL-скелет) сохраняется на диск с использованием базового кодирования JPEG (уровень качества 90\%). Это позволяет наглядно оценить физическую экономию дискового пространства.

4 Сбор статистики: вычисляются размеры оригинального, сжатого и восстановленного файлов (в килобайтах), а также рассчитывается фактический коэффициент компрессии в процентах. Полученные данные, совместно с путями к изображениям и рассчитанными метриками качества, передаются в HTML-шаблон для рендеринга итоговой страницы.

\subsection{Программная реализация вейвлет-кодера}

Модуль декомпозиции и сжатия реализован в классе WaveletCompressor с использованием библиотеки PyWavelets (pywt) и библиотеки компьютерного зрения OpenCV (cv2).

Алгоритм обработки отдельного изображения включает следующие функциональные этапы:

1 Подготовка тензора: загруженное изображение конвертируется в массив чисел с плавающей точкой (numpy.float32). Это критически важно для предотвращения потери точности и возникновения ошибок переполнения (overflow) в процессе вычисления сверток.

2 Разделение цветовых каналов: изображение разбивается на независимые цветовые плоскости (B, G, R). Двумерное дискретное вейвлет-преобразование pywt.dwt2 с базисом haar применяется итеративно к каждому из каналов.

3 Нормализация коэффициентов: математическая реализация фильтра Хаара в библиотеке PyWavelets приводит к увеличению значений амплитуды аппроксимирующего поддиапазона (LL) пропорционально коэффициентам фильтра. Для возвращения пикселей к оригинальному диапазону яркости, значения матрицы LL принудительно делятся на 2.0.

4 Квантование и сборка: нормализованные матрицы аппроксимации для каждого канала объединяются (cv2.merge). Для устранения вычислительного шума применяется математическое округление (np.round), после чего значения жестко ограничиваются в диапазоне $[0, 255]$ функцией np.clip и приводятся к целочисленному типу uint8.

На выходе формируется валидное растровое изображение, геометрические размеры которого ровно в два раза меньше исходных.

\subsection{Интеграция модуля нейросетевого апскейлинга}

Задачу реконструкции высокочастотных деталей выполняет класс AIEnhancer. Для обеспечения максимальной производительности при инференсе нейронной сети используется модуль dnn\_superres, встроенный в библиотеку OpenCV.

Функциональная схема модуля декодера:

1 Загрузка весов модели: при инициализации класса алгоритм проверяет наличие файла сериализованного графа модели EDSR\_x2.pb. В случае его отсутствия запускается автоматическое скачивание актуальных весов из удаленного репозитория.

2 Инициализация суперразрешения: создается объект DnnSuperResImpl, в который загружается вычислительный граф. Модели явно задается архитектура (edsr) и коэффициент масштабирования (scale = 2).

3 Инференс: сжатое изображение передается в метод upsample. Библиотека OpenCV автоматически выделяет тензоры, проводит данные через все сверточные слои EDSR и выполняет финальную субпиксельную перестановку (PixelShuffle), возвращая увеличенное изображение.

4 Постпроцессинг геометрии: ввиду особенностей целочисленного деления при вейвлет-декомпозиции нечетных разрешений, на этапе восстановления возможна размерная нестыковка (микро-сдвиги на 1 пиксель). Алгоритм выполняет программную валидацию геометрии: если размер выходного тензора не совпадает с оригинальным, применяется корректирующее интерполирование (cv2.resize).

\subsection{Реализация алгоритма метрического контроля}

Процесс объективной оценки качества реконструкции локализован в отдельном программном скрипте и базируется на двух функциях.

Расчет пикового отношения сигнала к шуму (PSNR) выполняется стандартными средствами OpenCV (cv2.PSNR), которые эффективно обрабатывают многоканальные (BGR) изображения.

Для вычисления индекса структурного сходства (SSIM) применяется функция structural\_similarity из библиотеки scikit-image (skimage.metrics). Поскольку алгоритм вычисления базового SSIM оперирует изменениями локальной контрастности и дисперсии, перед передачей тензоров в функцию оригинальное и восстановленное изображения в обязательном порядке преобразуются из цветового пространства BGR в градации серого (cv2.COLOR\_BGR2GRAY).

Интеграция данных модулей формирует законченную функциональную схему гибридного кодека, реализующую полный цикл от загрузки файла до метрической верификации нейросетевой реконструкции.
