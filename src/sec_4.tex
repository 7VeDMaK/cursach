\section{Разработка алгоритмов работы гибридного кодека и их программная реализация}

\subsection{Архитектура контроллера и программная маршрутизация данных}

Программная реализация гибридного кодека выполнена на языке Python 3 с использованием микрофреймворка Flask для организации серверной логики и веб-интерфейса. Выбор Python обусловлен наличием стандартизированной экосистемы для компьютерного зрения (OpenCV) и научных вычислений (NumPy).

Функционально серверная часть приложения (файл \texttt{app.py}) выступает в роли контроллера (Controller) и разделена на логические блоки, обеспечивающие полный цикл обработки:
\begin{enumerate}
    \item \textbf{Инициализация окружения:} при запуске сервера создаются директории для временного хранения загружаемых оригиналов (\texttt{UPLOAD\_FOLDER}) и сгенерированных результатов (\texttt{RESULTS\_FOLDER}). В оперативной памяти однократно инициализируются объекты классов \texttt{WaveletCompressor}, \texttt{AIEnhancer} и \texttt{MetadataEngine}.
    \item \textbf{Обработка ввода и кэширование:} контроллер принимает HTTP POST-запрос с массивом графических файлов. Для обеспечения удобства проведения лабораторных испытаний реализован механизм кэширования: если файлы не загружены повторно, контроллер считывает пути из скрытого поля \texttt{cached\_files} и обрабатывает их с новыми гиперпараметрами.
    \item \textbf{Программная маршрутизация (Smart Router):} с помощью функции \texttt{os.path.getsize} вычисляется физический объем файла. Если он меньше 50 Кбайт, активируется ветвление \textit{Pass-Through}, и файл копируется без изменений. В противном случае контроллер вычисляет количество мегапикселей и автоматически назначает уровень DWT-декомпозиции (переменная \texttt{comp\_level}).
    \item \textbf{Сбор статистики и тайминг:} с помощью библиотеки \texttt{time} алгоритм раздельно фиксирует время кодирования ($t_{enc}$) и декодирования ($t_{dec}$). Рассчитывается фактический коэффициент компрессии. Данные передаются в HTML-шаблон (движок Jinja2) для рендеринга итоговой страницы.
\end{enumerate}

\subsection{Программная реализация каскадного вейвлет-кодера}

Модуль декомпозиции и сжатия реализован в классе \texttt{WaveletCompressor}. Для обеспечения экстремально высокой скорости кодирования на мобильных (Edge) устройствах, математическая экстракция аппроксимирующей DWT-субполосы (LL) реализована через аппаратные примитивы библиотеки OpenCV.

Алгоритм обработки (\texttt{cascade\_compress}) включает следующие этапы:
\begin{enumerate}
    \item Входящее изображение в формате BGR конвертируется в многомерный тензор NumPy.
    \item На основе вычисленного уровня сжатия (Level 1, 2 или 3) алгоритм вычисляет фактор децимации: $scale = 2^{level}$.
    \item Итеративное отсечение высокочастотных фильтров заменяется однопроходной операцией субдискретизации по площади (\texttt{cv2.resize} с флагом интерполяции \texttt{cv2.INTER\_AREA}). Данная функция математически эквивалентна усреднению соседних пикселей в окне $2 \times 2$, что полностью повторяет расчет LL-квадранта вейвлета Хаара, но выполняется на уровне скомпилированного C++ кода OpenCV, ускоряя процесс в десятки раз.
    \item Полученный «скелет» сохраняется на файловой системе с использованием функции \texttt{cv2.imencode} в формат JPEG с жестко заданным параметром качества \texttt{cv2.IMWRITE\_JPEG\_QUALITY} равным 85. Это обеспечивает эффективное энтропийное кодирование DWT-матрицы.
\end{enumerate}

\subsection{Реализация модуля экстракции гибридных метаданных}

Извлечение структурно-цветовых векторов реализовано в классе \texttt{MetadataEngine} и выполняется параллельно с вейвлет-декомпозицией.
\begin{enumerate}
    \item \textbf{Экстракция цвета:} метод \texttt{extract\_color\_palette} динамически высчитывает размер паспорта (5\% от оригинала, но не более $150 \times 150$ пикселей). Выполняется сжатие (\texttt{INTER\_AREA}) для усреднения хроматики. В случае активации ручного режима (Manual Mode) размер может быть жестко задан параметром \texttt{force\_size}.
    \item \textbf{Экстракция контуров:} метод \texttt{extract\_edge\_map} конвертирует изображение в градации серого (\texttt{cv2.cvtColor(..., COLOR\_BGR2GRAY)}) и применяет оператор \texttt{cv2.Canny} с порогами 100 и 200. Полученная бинарная маска сжимается с коэффициентом $0.25$ или $0.5$ в зависимости от исходного разрешения.
\end{enumerate}

\subsection{Программная реализация нейросетевого апскейлинга и тайлинга}

Задачу реконструкции высокочастотных деталей выполняет класс \texttt{AIEnhancer}. Инференс моделей осуществляется через подмодуль \texttt{dnn\_superres}, встроенный в OpenCV.
\begin{enumerate}
    \item \textbf{Инициализация DNN:} создается объект \texttt{DnnSuperResImpl\_create()}. В него загружается сериализованный граф модели (\texttt{ESPCN\_x2.pb} или \texttt{FSRCNN\_x2.pb}). Модели настраиваются на использование CPU (\texttt{DNN\_TARGET\_CPU}), что гарантирует кроссплатформенную работу сервера без требования наличия дискретного GPU.
    \item \textbf{Тайловая маршрутизация:} ввиду ограничений RAM, реализованы методы \texttt{extract\_patches} и \texttt{merge\_patches}. Изображение разбивается на блоки заданного размера (\texttt{patch\_size = 256}) с нахлестом (\texttt{overlap = 16}). Это реализуется путем итеративного слайсинга (среза) NumPy-массивов.
    \item \textbf{Инференс и склейка:} каждый тайл проходит через функцию \texttt{sr.upsample()}. После возврата тензора, краевые 16 пикселей отсекаются для предотвращения граничных артефактов свертки, а оставшиеся центральные части копируются в итоговую пустую матрицу (\texttt{np.zeros}), размер которой предварительно рассчитан.
\end{enumerate}

\subsection{Семантическая постобработка и алгоритмы метрического контроля}

Финальный этап рекомбинации выполняется методами класса \texttt{MetadataEngine}.
Переданный цветовой паспорт масштабируется до размеров восстановленного изображения высокоточной интерполяцией \texttt{INTER\_LANCZOS4}. Затем применяется функция \texttt{cv2.addWeighted}, выполняющая линейное смешивание (альфа-блендинг) с коэффициентом 0.85 для цветовой карты и 0.15 для тензора яркости.

Для повышения резкости (\texttt{apply\_edge\_sharpening}) формируется нерезкая маска: изображение размывается функцией \texttt{cv2.GaussianBlur}, после чего из оригинала вычитается размытая версия. Результат накладывается на изображение поэлементным матричным умножением, где в качестве ограничивающего множителя выступает нормализованная карта контуров Кенни. Значения ограничиваются в диапазоне $[0, 255]$ функцией \texttt{np.clip}.

Процесс объективной оценки качества реконструкции локализован в файле \texttt{metrics.py}.
\begin{enumerate}
    \item Расчет \textbf{PSNR} выполняется функцией \texttt{cv2.PSNR(original\_img, restored\_img)}.
    \item Для вычисления \textbf{SSIM} применяется функция \texttt{structural\_similarity} из библиотеки \texttt{skimage.metrics}. Для корректной оценки структурных искажений, тензоры предварительно конвертируются в цветовое пространство \texttt{BGR2GRAY}.
\end{enumerate}

Интеграция данных модулей формирует законченную программную архитектуру асимметричного гибридного кодека, реализующую полный цикл от загрузки файла до семантической постобработки и метрической верификации.
