\section{Выбор и обоснование алгоритма работы проектируемой системы}

\subsection{Обоснование общей архитектуры программного комплекса}

Проектируемая система реализует гибридный асимметричный алгоритм сжатия и восстановления цифровых изображений, сочетающий классические методы цифровой обработки сигналов и технологии глубокого машинного обучения. В основу системы положена клиент-серверная архитектура, реализующая концепцию Edge-to-Cloud.

Данная архитектура позволяет перенести основную вычислительную нагрузку с маломощного устройства-отправителя на высокопроизводительный сервер. Структурная схема системы, отражающая взаимодействие модулей и потоки данных, представлена на рисунке~\ref{fig:structural_scheme}.

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.9\textwidth]{media/images/structural_scheme.png}
    \caption{Структурная схема гибридного программного комплекса}
    \label{fig:structural_scheme}
\end{figure}

В функциональный состав системы входят следующие модули:
\begin{enumerate}
    \item Модуль адаптивной маршрутизации: выполняет первичный анализ метаданных файла (размер, разрешение) и выбирает оптимальный путь обработки: сквозную передачу или гибридное сжатие.
    \item Модуль вейвлет-кодирования: реализует дискретное вейвлет-преобразование (DWT) Хаара для получения низкочастотной аппроксимации изображения, которая служит каркасом для передачи.
    \item Модуль экстракции метаданных: извлекает семантические дескрипторы (цветовую палитру и карту границ Кенни~\cite{canny1986}), которые весят единицы килобайт, но критически важны для предотвращения нейросетевых искажений.
    \item Модуль нейросетевого синтеза: на стороне приемника выполняет тензорный инференс моделей ESPCN~\cite{shi2016} или FSRCNN~\cite{dong2016fsrcnn}, восстанавливая текстуры и детали.
    \item Модуль постобработки и рекомбинации: выполняет комплексную коррекцию восстановленного результата в цветовом пространстве LAB для восстановления истинной хроматики и контраста.
\end{enumerate}

\subsection[Алгоритм работы модуля адаптивной маршрутизации и\protect\\ кодирования]{Алгоритм работы модуля адаптивной маршрутизации и кодирования}

Процесс кодирования спроектирован как последовательность легковесных операций, не требующих наличия графического процессора на стороне отправителя. Логика работы кодера представлена на рисунке~\ref{fig:encoder_algorithm}.

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.7\textwidth]{media/images/encoder_algorithm.png}
    \caption{Схема алгоритма адаптивной маршрутизации и кодирования}
    \label{fig:encoder_algorithm}
\end{figure}

Критическими этапами кодирования являются:
\begin{enumerate}
    \item Пороговая фильтрация: если объем данных мал (менее 50~Кбайт), алгоритм игнорирует сжатие, предотвращая ситуацию, когда метаданные весят больше самого изображения.
    \item Экстракция структурных контуров: для фиксации границ объектов используется детектор Кенни~\cite{canny1986}. Процесс включает гауссово размытие, вычисление градиента яркости оператором Собеля, подавление не-максимумов и двойную пороговую фильтрацию.
    \item Децимация через DWT: вместо простого изменения размера используется математически точная вейвлет-аппроксимация (LL-субполоса), что гарантирует сохранение энергетического баланса кадра.
    \item Контейнеризация: упаковывание вейвлет-скелета в формат JPEG (с коэффициентом качества 85) позволяет использовать эффективное энтропийное кодирование для минимизации объема передаваемых данных.
\end{enumerate}

\subsection{Алгоритм работы модуля нейросетевой реконструкции}

Модуль реконструкции выполняет наиболее ресурсоемкую часть работы, используя предобученные сверточные нейронные сети. Алгоритм декодирования представлен на рисунке~\ref{fig:decoder_algorithm}.

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.8\textwidth]{media/images/decoder_algorithm.png}
    \caption{Схема алгоритма нейросетевой реконструкции и постобработки}
    \label{fig:decoder_algorithm}
\end{figure}

Технологические особенности процесса реконструкции:
\begin{enumerate}
    \item Адаптивный выбор модели: система анализирует визуальную сложность входного кадра через дисперсию оператора Лапласиана. При высокой детализации используется модель FSRCNN~\cite{dong2016fsrcnn}, при низком шуме — ESPCN~\cite{shi2016}.
    \item Механизм тайловой маршрутизации: для обработки изображений высокого разрешения применяется нарезка на блоки с нахлестом 16~пикселей, что исключает возникновение видимых швов на границах.
    \item Двухэтапная коррекция в пространстве LAB: для устранения цветового сдвига и потери контраста применяется предварительное выравнивание средних значений каналов в нейросетевом модуле и последующий перенос статистических моментов (алгоритм Рейнхарда~\cite{reinhard2001}) по всем трем каналам (L, a, b) на основе цветового паспорта.
    \item Контурно-зависимое повышение резкости: фильтр нерезкой маски применяется с использованием мягкой маски прозрачности, сформированной путем гауссова размытия карты границ. Это позволяет локально усилить резкость контуров, не затрагивая однородные участки изображения.
\end{enumerate}

\subsection{Алгоритм оценки качества и профилирования}

Заключительный этап работы системы — объективный контроль качества. Алгоритм сопоставляет оригинальное и восстановленное изображения по следующим критериям:
\begin{enumerate}
    \item PSNR: определяет логарифмическую меру искажений на уровне пикселей.
    \item SSIM~\cite{wang2004ssim}: оценивает сохранность структурных паттернов, яркости и контраста текстур.
    \item Профилирование асимметрии: система фиксирует время кодирования и декодирования, подтверждая эффективность переноса вычислительной нагрузки на сервер.
\end{enumerate}
