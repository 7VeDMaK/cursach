\section{Выбор и обоснование структурной схемы (алгоритма работы) проектируемой системы}

\subsection{Обоснование общей архитектуры программного комплекса}

Проектируемая система реализует гибридный алгоритм сжатия и восстановления цифровых изображений, сочетающий методы классической цифровой обработки сигналов и глубокого машинного обучения. На основе проведенного анализа предметной области и сформированных требований, для реализации программного комплекса была выбрана клиент-серверная архитектура на базе паттерна MVC (Model-View-Controller).

Данный архитектурный подход обеспечивает четкое разделение логики вычислений, маршрутизации данных и пользовательского интерфейса. Структурно систему можно разделить на следующие функциональные модули:

1 Модуль маршрутизации и управления (Controller) — отвечает за прием исходных данных от пользователя, оркестрацию вызовов вычислительных подсистем и возврат результатов обработки.

2 Модуль вейвлет-компрессии (Кодер) — реализует алгоритмы цветовой конверсии и пространственно-частотной декомпозиции для уменьшения объема данных.

3 Модуль нейросетевой реконструкции (Декодер) — отвечает за инференс глубокой сверточной сети (EDSR) для масштабирования и восстановления деталей изображения.

4 Модуль оценки качества (Анализатор) — выполняет математическое сравнение оригинального и восстановленного изображений с вычислением объективных метрик.

Выбор веб-ориентированной архитектуры (на базе фреймворка) обоснован простотой развертывания: вся ресурсоемкая обработка (особенно инференс нейросети) инкапсулирована на стороне сервера, что позволяет использовать систему с любого клиентского устройства через стандартный веб-браузер без необходимости установки дополнительных библиотек.

\subsection{Алгоритм работы модуля вейвлет-компрессии}

Процесс сжатия графических данных на стороне сервера инициируется сразу после загрузки оригинального изображения. Для обеспечения максимальной эффективности декорреляции данных алгоритм переводит изображение из цветового пространства BGR (используемого библиотекой компьютерного зрения по умолчанию) в пространство YCrCb. Это позволяет отделить канал яркости (Y) от цветоразностных каналов (Cr, Cb).

После цветовой конверсии алгоритм выполняет независимую обработку каждого канала. Последовательность операций математического кодирования для отдельного канала выглядит следующим образом:

1 Инициализация базиса: выбирается ортогональный вейвлет Хаара (Haar wavelet) ввиду его вычислительной простоты и отсутствия краевых искажений при правильном симметричном дополнении (symmetric padding).

2 Пространственно-частотная декомпозиция: к двумерной матрице пикселей канала применяется прямое дискретное вейвлет-преобразование (DWT), в результате которого формируются четыре матрицы коэффициентов: аппроксимирующая (LL) и три детализирующие (LH, HL, HH).

3 Отсечение избыточности: детализирующие высокочастотные матрицы (LH, HL, HH), содержащие информацию о контурах и шуме, полностью удаляются из оперативной памяти. Дальнейшей обработке подлежит только матрица LL, пространственное разрешение которой ровно в два раза меньше оригинального.

4 Нормализация значений: поскольку вейвлет-коэффициенты матрицы LL могут выходить за границы стандартного 8-битного цветового диапазона, применяется линейная минимаксная нормализация (min-max scaling). Значения приводятся к целочисленному отрезку $[0, 255]$. Данный шаг вносит минимальную ошибку квантования, однако он критически важен для возможности сохранения промежуточного результата в стандартный графический формат.

По завершении обработки всех трех матриц, алгоритм объединяет нормализованные LL-субполосы обратно в трехканальный тензор, выполняет обратную конверсию из YCrCb в BGR и сохраняет результат в формате PNG (формат без потерь). В результате формируется файл, пространственное разрешение которого в 2 раза меньше оригинала (что эквивалентно аппаратному сокращению количества пикселей и объема данных в 4 раза). Полученный файл трактуется как «сжатое представление» (Compressed Image) и передается следующему модулю.

\subsection{Алгоритм работы модуля нейросетевой реконструкции}

Задачей модуля реконструкции является обратное преобразование: получение из сжатого низкочастотного представления (LL-скелета) полноразмерного изображения с восстановлением утраченных высокочастотных деталей. Поскольку классическое обратное вейвлет-преобразование (IDWT) при обнуленных коэффициентах детализации неспособно восстановить резкость контуров, система применяет алгоритмы машинного обучения.

Алгоритм декодирования базируется на использовании модуля глубоких нейронных сетей для сверхразрешения (DNN Super-Resolution). Логика работы модуля включает следующие шаги:

1 Загрузка вычислительного графа: в оперативную память загружается предварительно обученная архитектура сети EDSR (Enhanced Deep Residual Network), экспортированная в сериализованный бинарный формат графа. Выбор сети EDSR обусловлен её способностью обходиться без слоев пакетной нормализации (Batch Normalization), что повышает гибкость сети и экономит видеопамять;

2 Конфигурация параметров масштабирования: масштабирующий фактор сети (Scale Factor) жестко задается равным $r = 2$, что в точности соответствует коэффициенту децимации, примененному ранее алгоритмом DWT;

3 Инференс модели: сжатое изображение в формате BGR подается на входной слой нейронной сети. Данные проходят через каскад остаточных блоков (Residual Blocks), где сеть на основе выученных статистических закономерностей синтезирует недостающие текстуры. На финальном слое механизм субпиксельной свертки (PixelShuffle) увеличивает тензор до исходного разрешения;

4 Формирование результата: выходной тензор сети преобразуется в стандартный растровый формат и сохраняется на сервере как восстановленное изображение (Restored Image).

\subsection{Алгоритм оценки качества восстановления}

Для объективной оценки того, насколько успешно нейросетевой декодер справился с восстановлением информации, отброшенной вейвлет-кодером, в систему внедрен алгоритм автоматического метрического контроля. Анализ производится путем попиксельного сравнения оригинального изображения (до сжатия) и восстановленного изображения.

Вычисляются две фундаментальные метрики:

1 Пиковое отношение сигнала к шуму (PSNR) — логарифмическая метрика, оценивающая уровень среднеквадратичной ошибки (MSE) между изображениями. Чем выше значение PSNR (измеряется в децибелах, дБ), тем меньше физических искажений внес процесс сжатия-восстановления.

2 Индекс структурного сходства (SSIM) — метрика, учитывающая психовизуальное восприятие человека. Она оценивает изменения в структурной информации, яркости и контрасте. Значение метрики лежит в диапазоне от 0 до 1, где 1 означает абсолютную идентичность структуры. Алгоритм вычисляет SSIM для каждого цветового канала отдельно, а затем усредняет результат.

Вычисленные метрики, наряду с путями к сохраненным файлам (оригинал, сжатая версия, восстановленная версия), возвращаются контроллеру (модулю маршрутизации), который формирует итоговую веб-страницу для визуального анализа результатов пользователем. Предложенная структурная схема обеспечивает надежный, масштабируемый и математически обоснованный процесс компрессии графических данных.
